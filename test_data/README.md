# Test data — retail demo

Five related tables with short dummy data, ready for every supported connector.

| Table | Rows | Description |
|-------|------|-------------|
| `regions` | 3 | Sales regions |
| `customers` | 4 | Customer accounts |
| `products` | 4 | Product catalog |
| `orders` | 4 | Customer orders |
| `order_items` | 7 | Line items per order |

**Sample questions to try in chat:**
- How many orders does each customer have?
- What is total revenue by region?
- Which products appear most often in order lines?

---

## Quick start (SQLite only — no Docker)

```bash
source .venv/bin/activate
python sqlite_databases/seed.py --only retail
python test_data/test_connections.py --only sqlite
```

**In the Admin UI** (http://127.0.0.1:5001/admin):

1. **Add data source** → connector **SQLite**
2. **Database file:** pick `sqlite_databases/databases/retail.sqlite` (use absolute path; run `python sqlite_databases/seed.py` first)
3. Click **Test connection** → should succeed
4. **Import schema** → 5 tables
5. Go to **Chat** and ask a question

**Or register automatically** (backend must be running):

```bash
python test_data/register_via_api.py --only sqlite
```

## All connectors via Docker

Start Postgres + MySQL (and optionally SQL Server):

```bash
docker compose -f test_data/docker-compose.yml up -d postgres mysql
# optional, heavy: docker compose -f test_data/docker-compose.yml up -d mssql
```

Wait ~30s, then test adapters directly:

```bash
python test_data/test_connections.py --only postgresql
python test_data/test_connections.py --only mysql
python test_data/test_connections.py --only all
```

Register with the running app:

```bash
python test_data/register_via_api.py --only all
```

Connection presets live in `test_data/connections/`:

| File | Host | Port | User | Password |
|------|------|------|------|----------|
| `sqlite.retail.json` | file | — | — | — |
| `postgresql.retail.json` | localhost | 5433 | retail_reader | retail_reader |
| `mysql.retail.json` | localhost | 3307 | retail_reader | retail_reader |
| `mssql.retail.json` | localhost | 14333 | retail_reader | Retail_Reader1! |

Copy values from these JSON files into the Admin **Add data source** wizard, or use `register_via_api.py`.

## SQL Server note

SQL Server in Docker requires **ODBC Driver 18** on your Mac (`brew install unixodbc` + Microsoft driver). The MSSQL container may need manual init on first run:

```bash
docker compose -f test_data/docker-compose.yml up -d mssql
# then run test_data/schema/retail.mssql.sql via sqlcmd if tables are missing
```

If MSSQL is unavailable, SQLite + Postgres + MySQL are enough to prove the adapter abstraction.
